# 內容管理指南

## 概述

這個網站支援動態讀取 Markdown 檔案來生成電影評論和旅行筆記卡片。當您更新 `.md` 檔案時，網頁內容會自動更新。

## 電影評論管理

### 1. 創建 Markdown 檔案

在 `posts/movies/` 目錄下創建新的 `.md` 檔案，例如：`new-movie-2024.md`

### 2. 檔案格式範本

```markdown
# 電影標題

![電影海報](imgs/poster.jpg)

## 電影資訊
- **原名**: 英文或原文片名
- **導演**: 導演姓名
- **年份**: 2024
- **類型**: 動作片、科幻片
- **觀看日期**: 2024年1月1日
- **我的評分**: 4.5/5
- **推薦指數**: 4

## 劇情簡介

電影的劇情描述...

## 觀影心得

### 演技表現
演員的表現評論...

### 視覺效果
視覺效果的評價...

### 故事情節
劇情分析...

### 音樂配樂
配樂評論...

## 總結

最終評價和推薦...

---

*觀影筆記更新時間: 2024年1月1日*
```

### 3. 圖片處理規則

#### 圖片路徑規範

- 電影海報/劇照：放在 `posts/movies/imgs/` 目錄下
- 在 Markdown 中使用相對路徑：`![描述](imgs/filename.jpg)`
- 系統會自動轉換為正確路徑：`posts/movies/imgs/filename.jpg`

#### 支援的圖片格式

- JPG/JPEG、PNG、WebP、GIF
- 建議圖片寬度不超過 1200px
- 使用描述性檔案名稱

### 4. 自動解析欄位

| 欄位 | 格式要求 | 範例 | 說明 |
|------|----------|------|------|
| 標題 | `# 標題內容` | `# 復仇者聯盟` | 第一個一級標題 |
| 海報 | `![描述](路徑)` | `![海報](imgs/poster.jpg)` | 第一張圖片作為海報 |
| 原名 | `**原名**: 內容` | `**原名**: Avengers` | 原文片名 |
| 導演 | `**導演**: 內容` | `**導演**: 導演姓名` | 導演信息 |
| 年份 | `**年份**: YYYY` | `**年份**: 2024` | 發行年份 |
| 類型 | `**類型**: 內容` | `**類型**: 動作、科幻` | 電影類型 |
| 評分 | `**我的評分**: X.X/5` | `**我的評分**: 4.5/5` | 個人評分 |
| 推薦 | `**推薦指數**: X` | `**推薦指數**: 4` | 推薦等級 |

### 5. 更新配置檔案

編輯 `assets/config/content-config.js`，在 `MOVIE_FILES` 陣列中添加：

```javascript
{
    file: 'posts/movies/new-movie-2024.md',
    id: 'new-movie-2024',
    featured: true,  // 是否為精選內容
    images: ['assets/images/movie-poster.jpg'] // 可選：自定義海報路徑
}
```

## 旅行筆記管理

### 1. 創建旅行筆記檔案

在 `posts/travel/` 目錄下創建新的 `.md` 檔案，例如：`japan-trip-2024.md`

### 2. 旅行筆記格式範本

```markdown
# 旅行標題

![旅行照片](imgs/cover.jpg)

## 旅行資訊
- **日期**: 2024年6月17日 - 6月22日
- **地點**: 日本大阪
- **天數**: 6天5夜
- **交通方式**: 飛機+地鐵
- **住宿**: 心齋橋飯店
- **花費**: 約50000元
- **主要移動範圍路線**: 關西機場、心齋橋、中之島美術館

## 旅行心得

總體旅行感受...

### 印象深刻的景點

1. **大阪城** - 歷史建築，值得一訪
2. **心齋橋** - 購物天堂

### 美食體驗

- 大阪燒
- 章魚燒
- 神戶牛肉

## 旅行建議

- 提前訂票可以省錢
- 建議購買交通券
- 注意天氣變化

---

*更新時間: 2024年6月25日*
```

### 3. 旅行圖片管理

- 旅行照片：放在 `posts/travel/imgs/` 目錄下
- 使用相對路徑：`![描述](imgs/filename.jpg)`
- 支援多張圖片展示

### 4. 旅行資訊自動解析

系統會自動解析以下旅行資訊：

- **地點**: 旅行目的地
- **日期**: 旅行時間
- **天數**: 旅行天數
- **交通**: 交通方式
- **住宿**: 住宿信息
- **花費/預算**: 旅行費用
- **天氣**: 天氣情況
- **路線**: 主要行程路線

### 5. 旅行筆記配置更新

編輯 `assets/config/content-config.js`，在 `TRAVEL_FILES` 陣列中添加：

```javascript
{
    file: 'posts/travel/japan-trip-2024.md',
    id: 'japan-trip-2024',
    featured: true,
    images: ['posts/travel/imgs/japan-cover.jpg'] // 可選：自定義封面圖
}
```

## 進階功能

### 1. 目錄自動生成

系統會根據 Markdown 標題（h1-h6）自動生成：

- 左側導航目錄
- 平滑滾動到對應章節
- 標題層級縮排
- 懸停交互效果

### 2. 響應式圖片

所有圖片自動支援：

- 自動寬高比檢測（橫式/直式/方形）
- 響應式顯示調整
- 載入錯誤處理
- 圓角陰影效果

### 3. 智能卡片顯示

- 無內容時自動隱藏資訊卡片
- 動態標題更新
- SEO 友好的頁面標題

## 評分標準建議

### 電影評分 (1-5分制)

- **5分**: 傑作，強烈推薦
- **4分**: 優秀，值得觀看  
- **3分**: 普通，可看可不看
- **2分**: 一般，不太推薦
- **1分**: 糟糕，不推薦

### 推薦指數 (1-5級制)

- **5級**: 必看推薦
- **4級**: 高度推薦
- **3級**: 一般推薦
- **2級**: 謹慎推薦
- **1級**: 不推薦

## 注意事項

1. **檔案編碼**: 必須使用 UTF-8 編碼
2. **路徑格式**: 圖片路徑使用正斜線 `/`，避免反斜線 `\`
3. **格式一致**: 保持資訊區塊格式的一致性
4. **配置同步**: 每次新增檔案都需更新 `content-config.js`
5. **瀏覽器快取**: 更新後如未生效請清除瀏覽器快取

## 測試與調試

1. 開啟瀏覽器開發者工具檢查控制台錯誤
2. 確認檔案路徑是否正確
3. 驗證 Markdown 格式是否符合規範
4. 測試響應式顯示效果

完成添加後，訪問 `http://localhost:13145` 檢查新內容是否正確顯示。
